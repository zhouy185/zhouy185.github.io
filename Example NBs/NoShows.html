The Problem¶

In healthcare systems, it is common that patients may not show up for their appointments, causing waste of resources and unnecessary waiting times for other patients.

  • Can we predict if a patient will show up?

Question: What may be a possible challenge for the classification problem?

The dataset is unbalanced!

  • Many more No's than Yes's
  • If we simply predict that any patient will show up (i.e., No no-show), then we will have a relatively good accuracy.

Dealing with unbalanced dataset¶

What should we do?

  • Under-sampling

    • Select and include a subset of records of the majority class, so that we have about the same number of majority-class recods and minority-class records.

  • Over-sampling

    • Increase the number of records of the minority class, by doing sampling with replacements.

Use under-sampling when:

  • You have a large dataset, and removing some samples will not cause a significant loss of information.
  • You want to speed up training and reduce memory consumption.

Use over-sampling when:

  • Your dataset is small, and removing majority class samples would significantly reduce information.
  • You want to avoid discarding any data points.
  • You have enough computational resources to handle the increased dataset size.

One can also do "hybrid-sampling". How?

Feature Scaling¶

What is feature scaling?¶

A technique used to standardize or normalize the range of independent variables (features) in a dataset.

In [9]:
import matplotlib.pyplot as plt
plt.boxplot([appts['LeadTime'],appts['Age']])
plt.xticks([1,2],['Lead time','Age'])

We will perform standard scaling of the features.

For example, consider the Age column.

  • Its mean is 36, and the standard dev. is 22.

Why scaling the features?¶

  • For models like linear regression and logistic regressions, scaling helps in achieving faster and more stable convergence when training the models.
  • For distance-based models like K-Nearest Neighbors (KNN), features with larger scales can disproportionately influence these distances, leading to biased results.

How to scale features?¶

To implement the standard scaler, we need to import the StandardScaler from sklearn. Then, we need to:

  1. Create the scaler
  2. Fit it to the feature data: This computes the mean and standard deviation for each feature in the dataset.
  3. Transform the data: This scales the feature data

We will only scale LeadTime and Age (why?).

The Logistic regression model¶

Recall the linear regression suppose that the target variable is a linear function of the predictors

  • $f(x_1,\ldots,x_n)=b_0+b_1 x_1 +b_2 x_2 + \cdots + b_n x_n$, which is sometimes called the decision function

But it may not be ideal for predicting the class of a data point. Why?

We can use $f(x_1,\ldots,x_n)$ to model the probability of a patient with the features $x_1,\ldots,x_n$ being in class 1, i.e., a no-show.

$$\mathbb{P}(y=1)=\frac{e^{f(x_1,\ldots,x_n)}}{e^{f(x_1,\ldots,x_n)}+1}$$

To implement the logistic regression mode, let's define our features (predictors) and our target variables.

  • Drop the target variable No-show along with PatientId, AppointmentID and AppointmentDay to form the feature data X.
  • Use the No-show variable as the target data y.

Feature importance:

  • The coefficients measure by how much the decision function (and thus the probability of being in class 1) will change if the feature changes by 1 unit
  • In the current example,
    • how do the features affect the outcome, and
    • which feature is more important/influential?

Note: Feature importance, the way it was defined here, relies on the unit of measure of each variable. It is important to note that the importance levels derived are only for the case where variables are in their current units of measure (because it involves the change in the value of the variable by one unit), and are not representing a general evaluation of features, regardless of their unit of measure.

The ROC curve and the AUC score¶

The receiver operating curve plots:

  • the True Positive Rate (Recall) on the y-axis, vs.
  • the False Positive Rate on the x-axis,

for various probability thresholds.

To plot the ROC curve, we will use the function roc_curve in the sklearn.metrics module to get false positive rates (fpr) and false positive rates (tpr) for different values of thresholds.

  • The input arguments to the roc_curve are:
    • y_test, and
    • the probability of being in class 1 (no-show)
In [32]:
from sklearn.metrics import roc_curve, roc_auc_score

import matplotlib.pyplot as plt

fpr,tpr,thresholds = roc_curve(y_test,y_prob[:,1])

plt.plot(fpr,tpr)
plt.xlabel('False positive rate')
plt.ylabel('True positive rate')
plt.title('ROC curve')
plt.show()

The area under the curve measures overall ability of the model to distinguish between classes

  • The larger, the better performance of the model
  • Being greater than 0.5 means that it is better than random guesses

We can use the roc_auc_score function in the sklearn.metrics module to get the AUC score.

  • The input arguments are:
    • the observations in the test set (y_test)
    • the predicted probability of the positive class in the test set y_prob[:,1]
In [33]:

The definition of a "good" AUC score depends heavily on the specific domain or use case.

Rules of thumb:

  • a score 0.5 is the same as a random guess
  • a score of 0.7 or higher is often considered acceptable/adequate

Feature selection¶

Many predictors are included in our model. With a relatively small sample, this may potentially cause the problem of overfitting.

The main idea for mitigating overfitting is to reduce the complexity of the model.

  • For example, to use fewer features.

Recursive Feature Elimination (RFE)¶

RFE is a feature selection method in scikit-learn

  • It builds on top of a learning model.
  • It recursively removing features and building a model on the remaining features.
  • It ranks the features by importance and selects the top ones.

Example: Selecting 4 out of 6 features

Let's use RFE to select variables.

In [34]:
from sklearn.feature_selection import RFE

  • use estimator= to build the RFE model on top of the learning model
  • use n_features_to_select= to specify the number of features to include
  • use .fit() method of the RFE model to fit it to the training data
